Overview

This 4-stage CI/CD pipeline automates the process of building, testing, analyzing, and deploying applications using Jenkins, GitHub, SonarQube, Docker, & AWS EC2. The pipeline ensures efficient code integration, quality checks, and reliable deployment to end users.

Architecture Diagram

Pipeline Stages & Workflow

Stage 1: Source Code Management (GitHub)

Purpose: Store and manage code changes.

  1. Developers push code to a GitHub repository
  2. GitHub webhook automatically triggers Jenkins

Stage 2: Continuous Integration (Jenkins)

Purpose: Automate build and test processes.

  1. Jenkins pulls latest code from GitHub
  2. Executes build script and runs unit tests

Stage 3: Code Quality Analysis (SonarQube)

Purpose: Ensure code quality and security.

  1. SonarQube scans code for vulnerabilities
  2. Generates detailed quality report

Stage 4: Deployment (Docker & AWS EC2)

Purpose: Containerized deployment to cloud.

  1. Build Docker image and push to registry
  2. Deploy container to EC2 instance via SSH

Technology Stack

  • SonarQube

  • Nginx
